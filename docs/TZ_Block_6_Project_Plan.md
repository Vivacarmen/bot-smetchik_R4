# Блок 6: План проекта (Project Plan)

## 1. Общие принципы

- **AI-роль:** Senior Event Project Manager
- **Генерация:** Автоматически после финализации сметы (без подтверждения)
- **Режим:** Гибрид — AI предлагает задачи, менеджер редактирует одной кнопкой [✅ Принять / ✏️ Изменить]
- **Сроки:** AI сам оценивает сложность и назначает сроки, менеджер корректирует при необходимости
- **Подсветка:** AI помечает потенциально проблемные задачи, менеджер решает, красное это или нет

## 2. Структура задачи

| Поле | Тип | Ограничения |
|------|-----|-------------|
| ID задачи | INTEGER PK | Автоинкремент, уникальный в рамках плана |
| Название задачи | TEXT | Краткое, понятное |
| Описание | TEXT | Макс. 250 символов |
| Исполнитель/ответственный | TEXT | Свободный ввод |
| Контакт исполнителя | TEXT | Свободный текст (телефон/email/любое) |
| Дедлайн | DATE | Привязка к дате мероприятия (offset_days) |
| Приоритет | INTEGER | 1-5 (1 = низкий, 5 = критичный) |
| Категория/тип | TEXT | подготовка / день_мероприятия / закрытие |
| Зависимость от других задач | TEXT | ID задач-предшественников через запятую, или "0" |
| Проект/Смета ID | INTEGER FK | Ссылка на смету |
| Номер строки в смете | INTEGER | Строка в Excel-смете (0, если сложно определить) |

## 3. Категории задач

| Категория | Описание | Пример offset |
|-----------|----------|---------------|
| `подготовка` | До дня мероприятия | -10, -7, -3 дня |
| `день_мероприятия` | В день мероприятия | 0 |
| `закрытие` | После мероприятия | +1, +3 дня |

## 4. Генерация задач AI

### Логика создания

AI анализирует финальную смету и создаёт задачи по паттернам:

```
Услуга в смете → Задача(и) проекта

Примеры:
- "Аренда звука" → "Проверка звукового оборудования" (категория: подготовка)
- "Кейтеринг" → "Согласование меню с кейтерингом", "Контроль доставки оборудования"
- "Фото/видео" → "Брифинг фотографа", "Проверка резервного копирования"
```

### Расчёт сроков (offset_days)

AI оценивает сложность услуги и назначает срок:
- Стандартные услуги: -7 дней
- Сложные/заказные: -10...-14 дней
- Критичные (визы, разрешения): -21...-30 дней
- День мероприятия: 0
- Закрытие (отчёты, оплата подрядчиков): +1...+5 дней

### Зависимости между задачами

AI автоматически проставляет зависимости:
```
"Монтаж декорации" (ID 15) → зависит от → "Доставка декора" (ID 12)
"Репетиция звука" (ID 8) → зависит от → "Установка оборудования" (ID 5)
```

Если зависимость неочевидна — ставит `0`.

## 5. Подсветка "горящих" задач

AI помечает задачи флагом `⚠️ ТРЕБУЕТ ПРОВЕРКИ`, если:
- Срок менее 3 дней от сегодня
- Сложная услуга + короткий срок
- Нестандартное место проведения
- Первый раз работаем с подрядчиком

Менеджер вручную решает: оставить как есть, сдвинуть срок, или выделить красным в итоговом файле.

## 6. Комплекты документов по ролям

После финализации сметы генерируется:

| Роль | Комплект 1 (внешний) | Внутренние документы |
|------|----------------------|----------------------|
| Клиент/Агент | Договор, Приложение, Счёт, Акт (черновик) | — |
| Менеджер | Договор, Приложение, Счёт, Акт (черновик) | **План проекта (Excel)** + Тайминг мероприятия + Чек-лист координатора |
| Админ | Договор, Приложение, Счёт, Акт (черновик) | План проекта + Тайминг + Чек-лист + Сводка по рискам |
| CEO | Договор, Приложение, Счёт, Акт (черновик) | План проекта + Тайминг + Чек-лист + Сводка рисков + Смета с себестоимостью |
| Суперюзер | Договор, Приложение, Счёт, Акт (черновик) | План проекта + Тайминг + Чек-лист + Сводка рисков + Смета с себестоимостью |

### Внутренние документы

| Документ | Описание |
|----------|----------|
| **План проекта** | Excel с задачами (см. раздел 8) |
| **Тайминг мероприятия** | Пошаговый timeline дня мероприятия (чч:мм) |
| **Чек-лист координатора** | Список контрольных точек на площадке |
| **Сводка по рискам** | AI-анализ потенциальных проблем |
| **Смета с себестоимостью** | Полная копия с колонками cost_price (только CEO/Суперюзер) |

## 7. Тайминг мероприятия

Структура:
```
Время | Действие | Ответственный | Примечание
09:00 | Заезд на площадку | Технический директор | Пропуска
10:00 | Монтаж звука | Звукорежиссёр | Проверить розетки
11:00 | Монтаж декора | Декоратор | Фото до/после
...
```

AI генерирует на основе:
- Длительности мероприятия (из сметы)
- Сложности setup (из списка услуг)
- Опыт прошлых мероприятий (если есть)

## 11. Интеграция с Блоком 5 (Документооборот)

Последовательность:
1. Менеджер загружает финальную смету
2. Бот генерирует Комплект 1 (договор, приложение, счёт, акт)
3. **Автоматически** запускается генерация плана проекта
4. Менеджер получает: Комплект 1 + внутренние документы (включая план проекта)
5. AI-предложения задач: [✅ Принять все] [✏️ Редактировать] [🔄 Перегенерировать]

## 12. Что НЕ реализуется

| Функция | Причина |

| Gantt-диаграмма | Сложная визуализация, Excel достаточно |
| Автоматическая проверка выполнения | Только человек отмечает в трекере |
