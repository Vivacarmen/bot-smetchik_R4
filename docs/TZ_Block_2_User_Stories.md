# Блок 2: Сценарии пользователей (User Stories)

## 1. Роли и доступ

| Роль | Пароль | Что может | Особенности |
|------|--------|-----------|-------------|
| **Клиент** | Свой | Создавать сметы, редактировать свои, получать документы | Предупреждение о сохранении ФИО/телефона при первом входе |
| **Агент** | Свой | Создавать сметы для клиентов, редактировать свои, получать документы | Предупреждение о сохранении данных при первом входе |
| **Менеджер** | Свой | Создавать сметы, вносить расходы, управлять планами проектов, AI выступает как Senior Event PM | Основной оператор системы |
| **Админ** | Свой | Технический контроль: все сметы, настройки, бэкапы, блокировка пользователей | Ручное удаление данных по запросу |
| **CEO** | Свой | Бизнес-контроль: все данные, финансы, аналитика, редактирование чужих расходов | Нет технического доступа (удаление, raw SQL) |
| **Суперюзер** | Свой | Полный технический доступ: всё + удаление данных, редактирование любых записей, force logout | Техническая роль, не бизнес-роль |

### Различие CEO и Суперюзер

| Действие | CEO | Суперюзер |
|----------|-----|-----------|
| Видеть все сметы | ✅ | ✅ |
| Видеть финансы | ✅ | ✅ |
| Редактировать чужие расходы | ✅ | ✅ |
| Удалить смету/договор | ❌ | ✅ |
| Force logout пользователя | ❌ | ✅ |
| Доступ к raw SQL/БД | ❌ | ✅ |
| Изменить пароли ролей | ❌ | ✅ (через деплой) |

### Ограничения доступа

- **Срок сессии:** 8 часов с момента входа
- **Блокировка:** После 5 неудачных попыток — 15 минут
- **Повторная авторизация:** Только при истечении сессии

## 3. Команды по ролям

### 3.1 Общие команды (все роли)

| Команда | Описание | Доступ |
|---------|----------|--------|
| `/start` | Начать работу, выбор роли | Все |
| `/help` | Справка по командам роли | Все |
| `/my_smetas` | Мои сметы (с пагинацией) | Клиент, Агент, Менеджер |
| `/estimate` | Создать новую смету | Клиент, Агент, Менеджер |

### 3.2 Команды Менеджера

| Команда | Описание | Примечание |
|---------|----------|------------|
| `/estimate [ID]` | Открыть смету по ID | |
| `/finalize [ID]` | Финализировать смету | Запускает генерацию документов и плана проекта |
| `/expense` | Внести фактический расход | Блок 7.5 |
| `/my_expenses` | Мои расходы | |
| `/cost_report [ID]` | Отчёт "План vs Факт" по смете | Блок 7.5 |
| `/plan [ID]` | Просмотр плана проекта | Блок 6 |

### 3.3 Команды Админа

| Команда | Описание | Примечание |
|---------|----------|------------|
| `/all_smetas` | Все сметы в системе (пагинация) | |
| `/sync_calendar` | Принудительная синхронизация Google Calendar | Блок 8 |
| `/sync_yandex` | Принудительный бэкап на Яндекс.Диск | Блок 8 |
| `/test_webhook` | Тест webhook | Блок 8 |
| `/api_status` | Статус API (включено/выключено) | Блок 8 |
| `/security_logs` | Просмотр security-логов | Блок 9 |
| `/block_user [ID]` | Заблокировать пользователя | Блок 9 |
| `/unlock_user [ID]` | Разблокировать пользователя | Блок 9 |
| `/delete_data [ID]` | Ручное удаление данных клиента | По запросу, с подтверждением |

### 3.4 Команды CEO

| Команда | Описание | Примечание |
|---------|----------|------------|
| `/analytics` | Сводная аналитика (все бренды) | Блок 7 |
| `/cost_report [ID]` | Себестоимость любой сметы | Блок 7.5 |
| `/expense_edit [ID]` | Редактировать любой расход | Блок 7.5 |
| `/all_expenses` | Все расходы компании | Блок 7.5 |

### 3.5 Команды Суперюзера

| Команда | Описание | Примечание |
|---------|----------|------------|
| `/force_logout [ID]` | Принудительный выход пользователя | Блок 9 |
| `/expense_delete [ID]` | Удалить расход (soft delete) | Блок 7.5 |
| `/raw_sql` | Прямой доступ к БД (только чтение) | Опасно, логируется |
| `/system_status` | Полный статус системы | |

### 3.6 Служебные команды (для разработки)

| Команда | Доступ | Описание |
|---------|--------|----------|

| `/hotfix` | CEO | Срочный hotfix |

**Примечание:** GitFlow-команды — подсказки, реальное выполнение через GitHub.

## 5. Edge cases (дополненные)

| Ситуация | Реакция системы |
|----------|---------------|
| Попытка просмотра чужой сметы | ❌ Доступ запрещён + алерт в security_logs |
| Истечение сессии во время работы | ⚠️ "Сессия истекла. Введите пароль:" |
| Блокировка после 5 попыток | 🚫 15 минут + уведомление @vivacarmen |
| Попытка CEO удалить смету | ❌ "Удаление доступно только Суперюзеру" |
| Попытка Суперюзера редактировать без лога | ❌ Невозможно, лог обязателен |

## 7. AI-роли в сценариях

| Роль AI | Где применяется | Функция |
|---------|---------------|---------|
| **Сметчик** | Создание сметы | Расчёт цен, подбор услуг |
| **Senior Event PM** | План проекта (Блок 6) | Генерация задач, оценка сроков, выявление рисков |
| **Аналитик** | Аналитика (Блок 7) | Тренды, аномалии, рекомендации |
| **Помощник** | Везде | Ответы на вопросы, подсказки |

**Примечание для Менеджера:** AI в роли Senior PM предлагает задачи, но финальное решение — за менеджером (гибридный режим, Блок 6).
