# Блок 7: Аналитика

## 1. Общие принципы

- **Уровень доступа:** Компанийский — CEO/Админ/Суперюзер видят всю компанию, все бренды
- **Периодичность:** По запросу (команда `/analytics` или кнопка)
- **Формат:** Excel-файл для скачивания
- **Визуализация:** Только таблицы, графики не нужны
- **Обновление данных:** Только при запросе отчёта (не кешируем, не фоновые job)

## 2. Типы отчётов

### 2.1 Операционные отчёты

| Метрика | Расчёт | Примечание |
|---------|--------|------------|
| Количество созданных смет | COUNT за период | По дате создания |
| Количество финализированных смет | COUNT статус = final | |
| Количество договоров | COUNT из таблицы contracts | |
| Конверсия смета → договор | (договоры / сметы) × 100% | % |
| Среднее время от сметы до договора | AVG(дата договора − дата сметы) | В днях |
| Количество мероприятий | COUNT проведённых | По дате мероприятия |

### 2.2 Услуговые отчёты

| Метрика | Расчёт | Примечание |
|---------|--------|------------|
| Популярность услуг | COUNT использования в сметах | ТОП-20 услуг |
| Динамика цен по услуге | AVG(base_price) по месяцам | Для одной услуги |
| Средний чек по типу мероприятия | AVG(total_client) | Группировка |
| Распределение бюджета | SUM по категориям услуг | Звук/свет/декор/кейтеринг... |

### 2.3 Финансовые метрики (обязательные)

| Метрика | Формула | Источник |
|---------|---------|----------|
| Общая выручка | SUM(total_client) | Сметы со статусом "договор" |
| Себестоимость | SUM(total_cost) | Только CEO/Суперюзер видят |
| Прибыль | Выручка − Себестоимость | |
| Средняя маржа (%) | (Прибыль / Выручка) × 100 | |
| Средний чек | AVG(total_client) | Все сметы |

## 3. Срезы данных (фильтры)

Отчёты строятся с обязательным фильтром:

| Параметр | Варианты | Примечание |
|----------|----------|------------|
| Бренд | R4 / Veloxy / IM / Все | Radio button |
| Период | Сегодня / Неделя / Месяц / Квартал / Год / Свой диапазон | |
| Статус сметы | Все / Черновик / Финальная / Договор | |

## 4. AI-анализ тренда

AI анализирует текущие данные и добавляет комментарий в Excel:

```
Пример вывода AI:
"Тренд: выручка R4 за март 2025 на 15% выше среднемесячной.
Причина: рост заказов на тимбилдинги (+3 сметы) и увеличение среднего чека.
Рекомендация: усилить продвижение корпоративов в апреле."
```

**Возможности AI:**
- **A.** Собирает данные, считает суммы
- **B.** Находит аномалии ("маржа ниже среднего на 30%")
- **C.** Даёт рекомендации ("Увеличьте цену на звук, спрос высокий")

## 6. Форматы экспорта

При запросе `/analytics` бот предлагает:

| Формат | Назначение | Структура |
|--------|------------|-----------|
| **Excel (.xlsx)** | Основной, с форматированием | 5 листов, формулы, цвета |
| **CSV** | Импорт в 1С/Бухгалтерию | Одна таблица, разделитель `;` |
| **PDF** | Для презентаций | Только Лист 1 (сводка), красивое оформление |
| **JSON** | Для API-интеграций | Массив объектов с метаданными |

## 7. Процесс запроса аналитики

### Шаг 1: Команда

```
/analytics
```

Или кнопка в админ-панели: [📊 Аналитика]

### Шаг 2: Выбор параметров

```

Статус смет: [Все] [Черновик] [Финальная] [Договор]

[🚀 Сформировать отчёт]
```

## 8. База данных

Таблица `analytics_queries` (лог запросов):

| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER PK | |
| user_id | INTEGER FK | Кто запросил |
| requested_at | TIMESTAMP | Время запроса |
| brand_filter | TEXT | R4/Veloxy/IM/all |
| period_from | DATE | Начало периода |
| period_to | DATE | Конец периода |
| status_filter | TEXT | Статус смет |
| file_path | TEXT | Путь к сгенерированному файлу |
| formats | TEXT | JSON массив ["xlsx", "csv", "pdf", "json"] |

## 9. Особенности доступа

| Роль | Доступ | Примечание |
|------|--------|------------|
| CEO | Полный доступ ко всем данным | Видит себестоимость, все бренды |
| Админ | Полный доступ ко всем данным | Видит себестоимость, все бренды |
| Суперюзер | Полный доступ ко всем данным | Видит себестоимость, все бренды |
| Менеджер | ❌ Нет доступа к `/analytics` | Только свои сметы в обычном режиме |

## 10. Интеграция с Блоком 8 (Интеграции)

Отчёты в формате CSV/JSON готовы для выгрузки:
- В CRM (Битрикс24, amoCRM)
- В 1С (бухгалтерия)
- В Google Sheets (для совместной работы)

API-эндпоинт (для будущего):
```
GET /api/v1/analytics?brand=R4&from=2025-03-01&to=2025-03-31&format=json
```

## 11. Что НЕ реализуется

| Функция | Причина |
|---------|---------|
| Автооповещения об аномалиях | Только по запросу |
| Графики и дашборды | Только таблицы в Excel |
| Фоновый пересчёт метрик | Только при запросе |
| Архивация старых данных | Всё хранится в SQLite |
| Доступ менеджеров к аналитике | Только CEO/Админ/Суперюзер |
